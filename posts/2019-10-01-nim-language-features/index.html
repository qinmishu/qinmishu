nim语言的特性笔记

2019-10-01•nim

从字符串获得字符

用low或high可以分别获得字符串或者数组的第一个元素和最后一个元素的索引。

var str:string="hello"

#the ways to get first and last char from an string 
#the char is always one byte
assert(str[0] == str[low(str)])
assert(str[str.len-1] == str[high(str)])

#iterate the char in a str. char is a 8bits unsigned int. 
for c in str:
    echo c

判断一个字符是否是空白字符

第一种方法是使用strutils中的find函数。第二种方法是判断元素是否在集合中。

import strutils,strformat 

# using the 'find' to check if a char is white space 
var white_spaces:string ="\t\n "
var ch:char = '\t'
if white_spaces.find('\t') >= 0:
  echo fmt"'{ch}' is a white space"

# using the 'in' to check if a char is white space 
var white_space_set:set[char]={'\t','\n',' '}
ch = '\t'
if ch in white_space_set:
  echo fmt"'{ch}' is in white space set"

16进制字符串转为10进制数字

在parseutils模块中有parseHex函数，可以用来将16进制字符串转为10进制数字。类似地，还有parseBin和parseOct函数用于2进制和8进制的转换。

在strutils模块中有fromHex, fromBin, fromOct函数可以完成类似功能。同时在其中还包括toHex, toBin, toOct函数来完成反向的转换。

import parseutils

var hexStr:string = "1B"
var num:int 
if parseHex(hexStr,num) > 0:
  echo "success:" & $num
else:
  echo "failure"

获取命令行参数

可以使用os模块的paramCount和paramStr来获取命令行参数的个数和每一个命令行参数。paramStr(0)可以用来获取应用程序的名称。paramStr(1)用来获取第一个参数，依此类推。和c语言的main函数的argc不同，如果没有指定任何命令行参数，paramCount为0。

import os,parseutils

var beepPerMinute = 60

if os.paramCount() <= 0:
  echo "usage: ./beep <beep per minute>"
  quit(-1)

if parseutils.parseInt(os.paramStr(1),beepPerMinute) <= 0:
  beepPerMinute = 60

var sleepTimeMillis:int = int(float64(60*1000)/float64(beepPerMinute))

var count = 1    
while true:
  echo $count & $chr(7)
  inc(count)
  os.sleep(sleepTimeMillis)

使用正则表达式

下面程序使用正则表达式来判断一个字符是否属于定义的合法字符的集合

import re
var legalChar = re"[a-zA-Z0-9\-_|:=; \/\(\)]"

echo match($'a',legalChar)
echo match($'9',legalChar)
echo match($'/',legalChar)
echo match($'(',legalChar)
echo match($')',legalChar)

多种从函数返回值的方式

第一种，函数最后一条表达式的值作为函数的返回值。

proc add1(a,b:int):int =
    a + b

echo add1(1,2)

第二种，使用函数的隐含返回变量result来返回值。

proc add2(a,b:int):int =
    result = a + b

echo add2(2,3)

第三种，通过return语句来返回值。

proc add3(a,b:int):int =
    return a + b

echo add3(3,4)

函数调用命名风格不敏感

nim程序的关键字和变量还是象c或java语言一样是大小写敏感的。比如下面的程序中if不能是IF，不能用A来访问a。

在调用函数的时候，对于同一个函数，却可以用不同的风格(camel case，flat case, snake case)去调用。比如，下面程序中的myProc可以用myProc()或myproc()或my_proc()三种方式来调用。

proc myProc()=
  echo "hello from my procedure"

var a = 1
if a == 1:
  myProc() #camel case
  myproc() #flat case
  my_proc() #snake case
  #kebab case like below is not supported
  #my-proc()

In most of the time, nim is case-insentive and style-insensitive. Suppose procedure myProc is defined, we can call it myProc,myproc or my_proc.

proc, func, method

proc用来定义静态绑定(static dispatch)的函数。

func用来定义静态绑定的纯函数，纯函数是指没有副作用的函数。纯函数的输出就是它的返回值，由它的输入参数决定。除了输入参数外，它不会使用任何全局变量。输入参数相同，它的返回值永远相同。

method用来定义动态绑定(dynamic dispatch)的函数。可以达到Java或C++这样的面向对象编程语言的多态(polymorphism)的效果。

单元测试

可以在被测的nim模块文件中通过doAssert在isMainModule条件编译块中来进行单元测试，如下：

when isMainModule:
  block:
    var jsondata = """{"username":"henry", "message":"hello world"}"""
    var msg:Message = parseMessage(jsondata)
    doAssert msg.username == "henry"
    doAssert msg.message == "hello world"
    echo("block 1 done")

也可以通过引入unittest模块来用其中定义好的模板(template)或宏(macro)来写单元测试用例。可以用的宏包括test, check和expect等。

# This is just an example to get you started. You may wish to put all of your
# tests into a single file, or separate them into multiple `test1`, `test2`
# etc. files (better names are recommended, just make sure the name starts with
# the letter 't').
#
# To run these tests, simply execute `nimble test`.

import unittest
import pspkg/nbnf/nbnf

test "parser peek":
  var p = newParser("abc")
  check p.peek().ch == 'a'
  check p.peek().ch == 'a'
  p.pos = 3 
  check p.peek().isEof
  check p.peek().isEof

test "parser escaped":
  check newParser("\\\"").escaped() == '\"'
  check newParser("\\n").escaped() == '\n'
  check newParser("\\t").escaped() == '\t' 
  check newParser("\\\\").escaped() == '\\'
  expect AssertionError:
    discard newParser("\\d").escaped()
